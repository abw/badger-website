# `sidebar`

The `sidebar` option is passed to the `SiteProvider` component to define items
to appear in the sidebar.

## Properties

The following properties can be defined.

<PropsTable
  items={[
    ['title',         '',     'An optional title for the top of the sidebar'],
    ['sections',      '',     'An array of sections in the sidebar'],
    ['menuClass',     'menu', 'A CSS class for the menus in the sidebar'],
    ['menuItemClass', 'item', 'A CSS class for the items in sidebar menus'],
  ]}
/>

## Sections

Each element in the `sections` array should be an object.  It can contain a
`title`, a `menu` and/or some additional `content`.  The `revealable` property
can be set `true` to make it a revealable details component. The `openPath`
property can be used to specify a base URL path. The menu will be revealed by
default for pages whose paths begin with that prefix.

<PropsTable
  items={[
    ['title',      '', 'A title for the section'],
    ['revealable', 'false', 'Boolean flag to indicate if the `menu` and/or `content` should be in a revealable details component'],
    ['openPath',  '', 'A URL prefix for pages for which the details should be revealed by default'],
    ['menu',  '', 'An array of menu items'],
  ]}
/>

## Menu Items

Each item in a `menu` should be an object containing a `to` property denoting
the path to the page and a `text` property for the corresponding text to be
displayed for the item. The `code` property can be specified as a an
alternative to `text` to render the item in a monospaced font.

An item in a `menu` can also have a nested `menu` array and an optional `title`
for that sub-menu.  If a `menu` property is present then any other properties
(except the optional `title` are ignored).
